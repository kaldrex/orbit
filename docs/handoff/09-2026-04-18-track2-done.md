## Lessons this session baked in (carry forward to Track 3)

1. **Log-first, retry-never.** When an HTTP call fails consistently, read the server logs before dropping batch size. I burned 4 import runs on this in Track 2.
2. **One path per problem.** Live streaming and historical backfill go through different code paths. The `/api/v1/raw_events` route is for 2-3 evt/s streaming; the `fast-copy-wacli-to-raw-events.mjs` is for one-shot bulk. Don't force both through the same pipe.
3. **Real data beats synthetic.** My wacli fixture had invented column names (`id`/`direction`/`body_preview`) that didn't match the real schema (`msg_id`/`from_me`/`text`). Before writing the importer, `.schema` the real DB.
4. **UTF-8 edge cases matter.** WhatsApp messages embed NULs (`\u0000`) and unpaired UTF-16 surrogates. Both break Postgres JSONB. `fast-copy` handles both; any future text-carrying writer should too.
5. **Plans + tests + evidence log = future-you's best friend.** Every design decision is in `docs/superpowers/specs/`; every implementation is in `docs/superpowers/plans/`; every claim has an artifact in `outputs/verification-log.md`. If a future session is confused, the spec > plan > log chain answers it.
