## What Orbit is

A founder's cross-channel relationship memory. The founder brings their own **OpenClaw agent runtime** on their own **claw** (a small VM). Claw owns the channel credentials (WhatsApp, Gmail, Calendar). Orbit is the memory layer they write observations into. Dashboard serves a constellation graph + per-person cards. Every SKILL on claw runs on the founder's own LLM budget; Orbit never holds an Anthropic key.

## The living loop

Timer #1 (Orbit): **pg_cron** decides schedules. Writes jobs to the `jobs` table.
Timer #2 (Claw): **systemd** polls `POST /api/v1/jobs/claim` every 15 min. Claims oldest queued job → dispatches to `dispatchers/<kind>.sh` → dispatcher invokes `openclaw agent --agent main --message "Run the orbit-<kind> skill…"` → SKILL calls CLI verbs + ONE Claude call + `orbit_observation_bulk` → dispatcher POSTs `/api/v1/jobs/report`.

**No developer SSH required** once claw is installed. Hardeep/Khushal/Chandan can bootstrap themselves via `orbit-observer-backfill` SKILL on their first tick.

## Non-negotiables (don't break these)

1. **API is the only writer.** Never write to `observations` / `persons` / any Orbit table from a client — go through `/api/v1/observations`.
2. **No Anthropic calls outside `orbit-claw-skills/**/SKILL.md`.** A regression test (`tests/unit/no-anthropic-outside-skills.test.mjs`) enforces this. It greps `scripts/`, `orbit-cli-plugin/`, `src/`, `orbit-rules-plugin/` for any `anthropic`/`claude-sonnet`/`claude-haiku`/`claude-opus` string. Fails CI if you reintroduce the anti-pattern.
3. **CLI is plumbing.** `orbit-cli-plugin` holds NO Anthropic key, runs NO LLM calls. If a verb needs judgment, push it into a SKILL.
4. **Umayr canary.** `GET /api/v1/person/67050b91-5011-4ba6-b230-9a387879717a/card` must byte-match `outputs/verification/2026-04-19-umayr-v0/card.json` on 5 core fields: name, category, company, title, relationship_to_me.
5. **Hard cutover.** Delete old code cleanly — no `// removed` comments, no `_unused` aliases, no back-compat shims.
6. **No git push without explicit user go.** Phase 6 is gated.

## What NOT to do

- Don't push to main — Phase 6 is gated.
- Don't write a Node script that calls Anthropic. Use a SKILL.
- Don't SSH to claw to "run enrichment manually." The daily cron does it. If it's broken, fix the SKILL or the job queue, not the founder-bypass path.
- Don't touch the canary baseline at `outputs/verification/2026-04-19-umayr-v0/`.
- Don't drop the migration tracker drift F1 fixed — `supabase/migrations/20260421_audit_cleanup.sql` is idempotent and captures the back-fill.

## Known debt (tracked, not blocking)

1. **OpenClaw agent freelances off-script on long Sonnet turns** — observed in F6's live run. Mitigations: tighten SKILL system prompts; or wrap the Claude call in a plumbing-only CLI verb that sends a fully-templated prompt.
2. **No automatic orphan-job reaper.** Jobs stuck in `claimed_at IS NOT NULL AND completed_at IS NULL` accumulate silently. Fix: a pg_cron job that runs `reap_orphan_jobs(interval '30 min')`.
3. **`capability_reports` table is empty.** Claw never POSTs an agent heartbeat. Onboarding polling UI spins forever. Fix: add one POST call to `~/orbit-job-runner/run-once.sh`.
4. **`/api/v1/persons/enriched` hits PostgREST 1000-row cap.** Dashboard shows 1,000 of actual higher count. Fix: cursor-paginate or jsonb aggregate.
5. **Aura Graph Analytics tier off.** `/api/v1/graph/communities` + `/centrality` removed in F2. Revisit if the tier is enabled; GDS code deleted cleanly so re-adding means restoring those two routes.
6. **wacli 0.5.0 on claw; 0.6.0 upstream is pure security/stability.** 15-min binary swap, no schema change. Not urgent.
